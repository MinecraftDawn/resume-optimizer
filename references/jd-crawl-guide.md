# 104 職缺爬蟲指引

此檔案說明如何使用 WebFetch 從 104 抓取職缺資料，以及 fallback 處理規則。

---

## A. 104 搜尋 URL 組成方式

### 基本格式

```
https://www.104.com.tw/jobs/search/?keyword={職稱關鍵字}&area={地區碼}&jobcat={職類碼}&ro=0&kwop=7&order=15&asc=0&page=1&mode=s&s9=1&langFlag=0
```

### 常用地區碼（area 參數）

| 地區 | 代碼 |
|------|------|
| 台北市 | 6001001000 |
| 新北市 | 6001002000 |
| 桃園市 | 6001005000 |
| 台中市 | 6001008000 |
| 台南市 | 6001011000 |
| 高雄市 | 6001014000 |
| 全台灣 | （省略此參數） |

### 常用職類碼（jobcat 參數）

| 職類 | 代碼 |
|------|------|
| 軟體工程師 | 2007001000 |
| 前端工程師 | 2007001004 |
| 後端工程師 | 2007001005 |
| 資料工程師/分析師 | 2007001012 |
| 產品經理 | 2007004000 |
| 行銷企劃 | 2006001000 |
| 業務 | 2001001000 |
| 人資 | 2015001000 |
| 財務/會計 | 2004001000 |
| UI/UX 設計師 | 2007005001 |

職類不確定時，僅用 keyword 搜尋，省略 jobcat 參數。

### 薪資篩選（選用）

在 URL 後加 `&salmin={最低月薪}&salmax={最高月薪}`，單位為千元：
- 例：月薪 4–6 萬 → `&salmin=40&salmax=60`
- 若不限薪資，省略此參數

---

## B. WebFetch 抓取方式

### 步驟一：抓取搜尋結果頁

使用 WebFetch 抓取搜尋結果 URL。104 搜尋結果頁為 React 渲染，但靜態 HTML 中仍包含基本 JSON 資料（`__INITIAL_STATE__` 或 LD+JSON 格式）。

**提取目標（從 HTML 中找出）：**
- `<div class="b-block__left">` 內的職缺卡片區塊
- 每個職缺卡片中：`data-job-no`（職缺代碼）、職稱、公司名稱、地點、薪資描述
- 職缺網址：從職缺代碼組成 `https://www.104.com.tw/job/{jobCode}`，並在搜尋結果與適配報告中以 Markdown 連結格式顯示
- 若 HTML 中含 `window.__INITIAL_STATE__` JSON → 優先從 JSON 解析，結構更完整

**若 WebFetch 回傳空白或 403/429：**
→ 進入 Fallback 流程（見 D 節）

### 步驟二：抓取個別 JD 頁面

```
https://www.104.com.tw/job/{jobCode}
```

**提取目標：**
- 職缺網址：`https://www.104.com.tw/job/{jobCode}`（由職缺代碼組成）
- 職稱（`<h1>` 或 `og:title`）
- 公司名稱
- 薪資範圍（`<p class="b-tag--default">` 月薪標籤）
- 職務描述（`<div class="job-description-table__data">`）
- 學歷/年資要求
- 技能標籤（`<div class="b-tag">`）
- 福利標籤

---

## C. 解析規則

### 若成功取得 HTML

1. 找出所有職缺卡片（通常在 `<article>` 或含 `job-` 的 div 內）
2. 每筆職缺提取：職稱、公司、薪資描述、工作地點、職缺代碼
3. 將職缺代碼整理成清單，供用戶確認要分析哪幾筆

### 若成功取得個別 JD 頁

從職務描述中提取：
- **硬性要求**：學歷、年資（以「N 年以上」識別）
- **技能關鍵字**：程式語言、工具、認證、方法論名稱
- **軟性要求**：溝通能力、主動積極等（標記為「軟性條件」，不計入技能評分）
- **薪資**：若有揭露，記錄為月薪範圍；若為「面議」，記錄為 null

---

## D. Fallback 處理規則

當 WebFetch 無法取得 104 頁面時（空白、渲染失敗、403），依序執行：

**Fallback 1 — 提供搜尋 URL 讓用戶自行訪問：**

> 「我無法直接抓取 104 頁面（可能需要登入或反爬蟲阻擋）。請打開以下連結，把你感興趣的職缺的完整網址（或 JD 文字）貼回給我：
> [URL]」

**Fallback 2 — 用戶貼入 JD 文字：**

若用戶貼入純文字 JD，依 C 節規則從文字中手動提取欄位。若職稱/公司/薪資欄位缺失，標記為 ❓，不影響評分（相應維度以中位數代入）。

**Fallback 3 — 用戶貼入多個 URL：**

對每個 URL 嘗試 WebFetch；成功者正常解析，失敗者要求用戶補充該職缺的 JD 文字。

---

## E. 抓取數量建議

- **預設**：抓取搜尋結果前 10 筆
- **用戶選擇分析**：建議 3–5 筆（完整適配分析）；最多 10 筆
- **單筆快速評估**：用戶直接提供 JD 連結或文字 → 省略搜尋步驟，直接進入 J3
