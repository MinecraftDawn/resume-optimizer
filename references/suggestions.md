# Optimization Suggestions Reference

## Section Suggestion Format

每個**達成率低於 75%** 的區塊，至少提供 2 條具體建議。格式如下：

```
### [區塊名稱] — X/滿分Y（達成率 Z%）  ⚠️/❌

**問題：** [具體弱點，引用履歷原文]

**建議 1：** [具體行動]
**建議 2：** [具體行動]
[XYZ 改寫示範（工作經歷/專案成就必附）]
```

---

## Achievement Rewrite: Google XYZ Formula

工作經歷與專案成就的每條任務型描述，套用 XYZ 公式改寫：

> **Accomplished [X] as measured by [Y], by doing [Z]**
> 中文版：「透過 [Z 方法]，達成 [X 成果]，以 [Y 指標] 衡量」

**改寫流程：**
1. 識別原句是「任務型」（負責 / 參與 / 協助）還是「成果型」（提升 / 降低 / 達成）
2. 任務型 → 詢問用戶：「這件事帶來什麼具體成果？有沒有數字？」
3. 若用戶提供數字 → 直接套入 XYZ 公式輸出改寫版
4. 若用戶無法提供數字 → 用「估算範圍」替代，並標注 `[待補數字]`

**改寫範例：**

| 原句（任務型） | 改寫後（XYZ 成果型） |
|--------------|-------------------|
| 負責社群媒體管理 | 管理 FB/IG 雙平台內容策略，3 個月自然觸及提升 40%、追蹤人數成長 2,000+，透過發文頻率優化與 UGC 活動執行 |
| 協助行銷預算規劃 | 優化月均 $50k 行銷預算配置，CPA 降低 15% 同時導流量成長 20%，透過 A/B 測試調整渠道比例達成 |
| 參與客戶服務流程改善 | 推動客服 SOP 重新設計，首次解決率從 62% 提升至 81%，透過客訴分類分析找出 Top 5 重複問題並建立標準回覆庫 |
| 負責後端 API 開發 | 設計並交付 RESTful API 服務，支撐日均 50 萬次請求，回應時間從 420ms 降至 95ms，透過 Redis 快取層與查詢優化實現 |

## JD Keyword Gap Analysis

**觸發條件：** 用戶在 Step 1 提供了目標職缺連結或 JD 文字。

### 執行流程

**Step A — 提取 JD 關鍵字**

從 JD 中萃取三類關鍵字：

1. **硬技能關鍵字**（工具、語言、平台、框架）
   - 例：Python、Google Analytics、Salesforce、React、SQL

2. **職能關鍵字**（職責動詞 + 職能名詞）
   - 例：「跨部門協作」、「數據驅動決策」、「客戶成功管理」、「A/B 測試」

3. **產業/域名關鍵字**（產業術語、職位特定詞）
   - 例：SaaS、B2B、電商、敏捷開發、GDPR、KYC

**Step B — 與履歷交叉比對**

對每個 JD 關鍵字，判斷履歷中的狀態：

| 狀態 | 定義 | 標記 |
|------|------|------|
| 完全匹配 | 履歷中有相同或同義詞，且有具體佐證 | ✅ |
| 弱匹配 | 履歷提到但缺乏具體化或語境不同 | ⚠️ |
| 缺失 | 履歷中完全未出現 | ❌ |

**Step C — 輸出 Gap 分析表**

```
### JD 關鍵字 Gap 分析

**目標職缺：** [職缺名稱]

| 類別 | JD 關鍵字 | 履歷狀態 | 建議行動 |
|------|----------|---------|---------|
| 硬技能 | Python | ✅ 完全匹配（工作經歷第2條） | — |
| 硬技能 | Tableau | ❌ 缺失 | 若有使用經驗，加入專長區塊 |
| 職能 | 跨部門協作 | ⚠️ 弱匹配（自傳提及但無佐證） | 在工作經歷中加入「帶領X人跨部門...」描述 |
| 產業 | SaaS | ❌ 缺失 | 若有 SaaS 產品經驗，在工作內容中補充 |

**匹配率：** X / Y 個關鍵字完全匹配（XX%）

**優先補強：** [列出最重要的 3 個缺失關鍵字，及建議放置位置]
```

**Step D — 調整工作經歷 JD 維度分數**

Gap 分析完成後，回到 `scoring.md` 的工作經歷評分，根據實際關鍵字匹配率更新「JD 關鍵字匹配」子項分數：

| 匹配率 | JD 維度分數（/10） |
|--------|-----------------|
| ≥80% | 9–10 |
| 60–79% | 7–8 |
| 40–59% | 5–6 |
| 20–39% | 3–4 |
| <20% | 0–2 |

## ATS 相容性檢查（台灣市場）

**觸發條件：** 用戶目標職缺為中大型企業（員工 200 人以上）、外商、或科技公司。

### 台灣常見 ATS 系統
- 104 履歷直接投遞：104 自有 ATS，使用 104 標準格式影響最小
- 企業自建/第三方 ATS（如 Greenhouse、Lever、Workable）：需注意格式相容
- 外商全球 ATS（如 Workday、SAP SuccessFactors）：對 PDF 格式最嚴格

### ATS 相容性 Checklist

依照以下清單檢查候選人履歷（若用戶提供 PDF，評估格式結構；若只有文字，跳過排版類項目）：

**✅ 建議做（ATS 友善）：**
- [ ] 使用標準區塊標題（工作經歷、學歷、技能，而非創意命名）
- [ ] 每個職位都有明確的公司名稱、職稱、起始時間
- [ ] 技能關鍵字直接出現在描述文字中（不只在標籤欄）
- [ ] 日期格式統一（如 2022/03–2024/06，而非「前年至去年」）

**❌ 避免（ATS 不友善）：**
- [ ] 表格或多欄位排版（ATS 通常無法正確解析）
- [ ] 重要資訊放在頁首頁尾（ATS 常忽略 header/footer）
- [ ] 圖片或圖形式技能評分（如圓形進度條）
- [ ] 嵌入式字型或特殊符號取代文字

**評分影響：** ATS 問題不計入現有評分表，但在「下一步行動清單」中列為優先提醒項，若有嚴重 ATS 問題則在報告開頭的總覽中特別標注。

## Common High-Impact Optimizations

### 個人特色 — 從通用到精準

- Weak: 積極、負責、細心
- Strong: React.js、數據分析、敏捷開發、B2B 業務、英文簡報

### 求職條件 — 期望工作內容策略

有明確 JD → 針對 JD 寫，展現匹配與貢獻：
- ✅ 「期望以數據驅動的行銷策略協助品牌成長，擅長 SEO/內容行銷，目標協助公司提升 organic traffic」

無明確 JD → 留空（保持機會廣度）或寫能力導向內容；避免通用願望清單：
- ✅ 留空：開放策略，不自我設限（合理選擇）
- ✅ 「擅長以 SEO/內容行銷驅動自然流量成長，能協助品牌在 organic 渠道建立長期競爭優勢」
- ❌ 「希望從事行銷相關工作」— 自我設限卻無任何價值

### 自傳開場 — 避免常見模板

- ❌ 「我叫XXX，畢業於OO大學...」
- ❌ 「我是一個積極進取、勇於挑戰的人...」
- ✅ 直接切入最強成就：「在過去 3 年的電商行銷工作中，我主導了 3 次品牌重新定位，每次都帶來雙位數的營收成長...」

### 語文能力 — 可信度建立

- ❌ 中文：精通、英文：精通、日文：精通（無任何佐證）
- ✅ 英文：工作應用（TOEIC 860，前職以英文撰寫所有技術文件）

### 專長可信度 — 交叉驗證

若宣稱某專長，確保工作經歷或專案中有至少一條具體使用記錄，否則：
- 移除該專長，或
- 在工作經歷中補充「使用 [工具/技能] 完成 [具體任務]」
