# JD 職缺適配評分 Rubric

此檔案定義「適配分析」模式（Step J1–J4）的評分維度、公式，與輸出格式。

---

## 評分架構（總分 100 分）

| 維度 | 滿分 | 說明 |
|------|------|------|
| 職稱 / 職類匹配 | 25 | 職稱相符程度、職類是否一致 |
| 技能關鍵字重疊率 | 30 | 履歷技能標籤 + 工作經歷關鍵字 vs JD 要求技能 |
| 年資符合度 | 20 | 履歷工作年資 vs JD 要求年資 |
| 薪資區間符合度 | 10 | 履歷期望薪資 vs JD 提供薪資（若有揭露）|
| 產業 / 地區符合度 | 15 | 履歷求職條件 vs JD 所在產業與地點 |

---

## 各維度評分規則

### 1. 職稱 / 職類匹配（25 分）

| 情境 | 分數 |
|------|------|
| 職稱完全相同（如：前端工程師 = 前端工程師） | 25 |
| 職稱高度相關（如：前端工程師 vs 全端工程師） | 18–22 |
| 職類相同但職稱不同（如：工程師 vs 資料分析師，同屬技術類） | 12–17 |
| 職類跨域但有遷移可能（如：行銷 vs 產品經理） | 6–11 |
| 職類明顯不符 | 0–5 |

**計算方式：**
1. 提取 JD 職稱
2. 比對履歷的目標職稱（求職條件欄位）與現職職稱
3. 依上表找出對應分數區間，取中位數；若有佐證（履歷中有明確相符經歷），取上限

---

### 2. 技能關鍵字重疊率（30 分）

**A. 提取履歷技能池（Resume Skill Pool）：**
從以下來源合併：
- 基本資料 → 個人特色標籤
- 每份工作經歷 → 工作技能標籤
- 工作內容描述中出現的工具/語言/框架名詞
- 專案成就中的技術描述

去重後得到一個技能集合 `R = {skill_1, skill_2, ...}`

**B. 提取 JD 技能要求（JD Skill Set）：**
從 JD 職務描述中識別：
- 硬技能：程式語言、框架、工具、認證、平台名稱
- 軟技能**不計入**（如「溝通能力」「主動積極」）

得到 JD 要求技能集合 `J = {skill_1, skill_2, ...}`，並標記：
- `J_required`：「必要」或「N 年以上」的技能
- `J_preferred`：「加分」「優先」的技能

**C. 計算重疊率：**

```
必要技能覆蓋率 = |R ∩ J_required| / |J_required|  （若 J_required 為空，此項設為 1.0）
加分技能覆蓋率 = |R ∩ J_preferred| / max(|J_preferred|, 1)

原始分 = 必要技能覆蓋率 × 24 + 加分技能覆蓋率 × 6
最終分 = min(原始分, 30)（上限 30）
```

**D. 關鍵字匹配注意事項：**
- 同義詞視為匹配：React / React.js、Python / Python3、AWS / Amazon Web Services
- 大小寫不敏感
- 若 JD 列出 3 項以下技能，技能維度最高得分設為 20（資訊不足）

---

### 3. 年資符合度（20 分）

**提取：**
- 履歷總工作年資 `R_exp`（計算所有工作經歷的年資總和，月轉年取一位小數）
- JD 要求年資 `J_exp`（格式通常為「N 年以上」；若未標明，視為無年資要求）

**評分規則：**

| 情境 | 分數 |
|------|------|
| JD 無年資要求 | 20 |
| `R_exp >= J_exp` | 20 |
| `J_exp - 1 <= R_exp < J_exp` | 15（差距 1 年以內）|
| `J_exp - 2 <= R_exp < J_exp - 1` | 10（差距 1–2 年）|
| `R_exp < J_exp - 2` | 5（差距超過 2 年）|
| `R_exp > J_exp + 5` | 16（過度資深，可能有薪資期待落差）|

---

### 4. 薪資區間符合度（10 分）

**條件：** 僅在 JD 揭露薪資範圍 **且** 履歷有填寫薪資期望時計算。

若任一方缺失 → 此維度得 **7 分**（中位數，不懲罰也不加分）。

**計算：**

設 `R_sal` = 履歷期望月薪（取中位數，如「4–6 萬」→ 5 萬），`J_sal_min`/`J_sal_max` = JD 薪資區間

| 情境 | 分數 |
|------|------|
| `J_sal_min <= R_sal <= J_sal_max` | 10 |
| `R_sal < J_sal_min`（履歷期望低於職缺下限） | 10（對用戶有利）|
| `J_sal_max < R_sal <= J_sal_max × 1.15` | 7（期望略高 15% 以內）|
| `R_sal > J_sal_max × 1.15` | 4（期望顯著超出）|

---

### 5. 產業 / 地區符合度（15 分）

**產業符合度（10 分）：**

| 情境 | 分數 |
|------|------|
| 履歷求職條件的目標產業 = JD 所屬產業 | 10 |
| 相鄰產業（如：科技業 vs 電商） | 7 |
| 跨產業但有遷移合理性 | 4 |
| 明顯跨域（如：製造業 vs 金融業） | 2 |
| 履歷無產業偏好 → | 7（中位數）|

**地區符合度（5 分）：**

| 情境 | 分數 |
|------|------|
| 地區完全符合 | 5 |
| 鄰近縣市（如：台北市 ↔ 新北市） | 4 |
| 同區域不同縣市（如：北部各縣市之間） | 3 |
| 跨區域（如：台北 ↔ 台中） | 1 |
| 願意全台 / 不限地區 | 5 |
| JD 允許遠端（Remote）→ 地區維度直接給 5 分 | 5 |

## 輸出格式

### 適配比較表（多 JD）

```
## 🎯 職缺適配分析結果

| 排名 | 職缺 | 公司 | 適配分 | 職稱符合 | 技能重疊 | 年資 | 薪資 | 產業/地區 |
|------|------|------|--------|---------|---------|------|------|---------|
| 🥇 1 | [職稱] | [公司] | **XX/100** | XX/25 | XX/30 | XX/20 | XX/10 | XX/15 |
| 🥈 2 | ... | ... | **XX/100** | ... | ... | ... | ... | ... |
| 🥉 3 | ... | ... | **XX/100** | ... | ... | ... | ... | ... |
```

### 個別 JD 詳細分析（前 3 名必做）

```
### 🏆 第 1 名：[職稱] @ [公司]
**總適配分：XX/100**　[████████░░]

**為什麼推薦：**
- [主要符合點 1]
- [主要符合點 2]

**主要技能 Gap（你有 vs JD 要求）：**
- ✅ 已有：[技能清單]
- ❌ 缺少：[技能清單]（建議補充優先度：高/中/低）

**投遞建議：**
[一句話說明投遞策略，如：技能高度匹配，薪資期待略高，建議先談 → 再議薪]
```

### 適配分數色階

| 分數 | 標示 | 意義 |
|------|------|------|
| 80–100 | 🟢 強力推薦 | 高度契合，優先投遞 |
| 60–79 | 🟡 值得嘗試 | 有缺口但仍具競爭力 |
| 40–59 | 🟠 有挑戰 | 需補強後再投，或先投其他職缺 |
| 0–39 | 🔴 不建議 | 目前條件落差過大 |
