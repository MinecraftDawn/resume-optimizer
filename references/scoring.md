# Scoring Reference

**總分：100 分（＋自訂內容最高加 3 分）**

每個區塊滿分依據對「實際篩選結果」的影響力設定，非平均分配。

| 區塊 | 滿分 | 重要性 |
|------|------|--------|
| 工作經歷 | 25 | ★★★★★ |
| 專長 | 15 | ★★★★ |
| 基本資料 | 10 | ★★★★ |
| 學歷 | 10 | ★★★ |
| 求職條件 | 8 | ★★★ |
| 自傳 | 8 | ★★★ |
| 專案成就 | 8 | ★★★ |
| 軟實力綜合 | 6 | ★★★ |
| 證照 | 4 | ★★ |
| 語文能力 | 3 | ★★ |
| 附件 | 2 | ★★ |
| 推薦人 | 1 | ★ |
| 自訂內容 | +3 | ★★ |

**職位加權說明：** 證照、語文能力、附件標示「×3」時，表示目標職缺明確需要該項，應在最終建議優先序中特別強調。

## 基本資料 (Basic Info) — 滿分 10

評估各子欄位內容質量，非填寫與否。

**個人簡介（權重 40%，內部評分 0–10）：**
| 分數 | 標準 |
|------|------|
| ≥8 | 職務導向，含具體技能/產業/年資，第一句是鉤子（例："5年電商行銷，專注 SEO 與 CRM"） |
| 6–7 | 有內容但通用，缺差異化 |
| 4–5 | 純軟實力宣言，無具體化 |
| 1–3 | 空白或僅一句話 |

**個人特色（權重 40%，內部評分 0–10，共 6 個標籤格）：**
| 分數 | 標準 |
|------|------|
| ≥8 | 6 個標籤全填；以硬技能/產業關鍵字為主 |
| 6–7 | 4–5 個標籤，硬軟混合 |
| 3–5 | 3 個以下標籤，或全是軟實力標籤 |
| 1–2 | 完全空白 |

**個人連結（權重 20%，內部評分 0–10）：**
| 分數 | 標準 |
|------|------|
| 8–10 | 連結與職務相關（工程師→GitHub，設計師→Behance），連結有效且更新 |
| 5–7 | 有連結但部分失效或不相關 |
| 2–4 | 無連結（技術/創意職缺建議補充，一般職缺尚可接受） |

**最終分：** `(簡介×0.4 + 特色×0.4 + 連結×0.2) / 10 × 10`

## 學歷 (Education) — 滿分 10

三因子組合。**先讀完所有因子再評分。**

**Factor 1 — 學位上限：**
| 學歷 | 分數上限 |
|------|---------|
| 博士 | 10 |
| 碩士 | 9 |
| 學士（4 年制） | 8 |
| 四技/二技（科技大學） | 7 |
| 專科（5/2 年制） | 6 |
| 高中/高職 | 5 |

**Factor 2 — 學校聲望調整：**
| 等級 | 代表學校 | 調整 |
|------|---------|------|
| S：頂尖國立 | 台大、清大、陽明交大、成大、政大 | +1.5 |
| A：優質國立 | 師大、中山、中央、中興、台北大、台科大、北科大 | +0.5 |
| B：知名私立/區域國立 | 輔大、淡江、東吳、文化、逢甲、中原、高科大 | +0 |
| C：一般大學 | 其他私立院校 | +0（學歷是過去的事，不額外扣分） |
| 海外 | QS 前 100 → +1.5；前 500 → +0.5；其他 → +0 | 依排名 |

*注意：理工職缺中，台科大/北科大可視同 A+ 等級（調整 +0.5 → +1.0）。*

**Factor 3 — 就學狀態係數：**
| 狀態 | 係數 |
|------|------|
| 畢業 | ×1.0 |
| 就學中（畢業日期明確） | ×0.9 |
| 肄業 | ×0.75 |

**計算：** `min(10, 學位上限 + 學校調整) × 狀態係數`

範例：
- 台大碩士畢業 → min(10, 9+1.5)×1.0 = **10**
- 逢甲大學畢業 → min(10, 8+0)×1.0 = **8**
- 中山大學就學中 → min(10, 8+0.5)×0.9 = **7.65 ≈ 7.5**
- 私立大學肄業 → min(10, 8+0)×0.75 = **6**

## 工作經歷 (Work Experience) — 滿分 25

評估完整職位內容，**非欄位填寫數量**。

| 評估維度 | 權重 | 內部評分 |
|---------|------|---------|
| 成果 vs 任務描述 | 40% | 0–10 |
| 職涯成長脈絡 | 20% | 0–10 |
| 工作技能標籤相關性 | 20% | 0–10 |
| 管理責任 | 10% | 0–10 |
| JD 關鍵字匹配（有JD才計） | 10% | 0–10 |

**成果 vs 任務（最重要）：**
| 分數 | 標準 |
|------|------|
| ≥8 | 多數描述有數字成果（"留存率從 60%→78%，年省 120 萬"） |
| 6–7 | 成果與任務混合，部分有數字，描述有具體感 |
| 4–5 | 純任務清單，有職責描述但缺乏衝擊數字 |
| 1–3 | 幾乎只有職稱與公司名，描述極少 |

**職涯成長脈絡：**
| 分數 | 標準 |
|------|------|
| ≥8 | 清晰晉升路徑，責任遞增 |
| 6–7 | 橫向移動有說明，或自然成長可見 |
| 4–5 | 跳槽頻繁（< 1 年/份）無說明，或明顯停滯 |
| 2–3 | 只有一份工作且無脈絡（對新鮮人屬正常情況） |

**新鮮人校準（< 1 年全職經驗）：**
職涯成長脈絡不適用時，改評估「學習軌跡」：
| 分數 | 標準 |
|------|------|
| ≥7 | 有實習/打工/side project 連貫脈絡，方向一致 |
| 5–6 | 有經驗但片段，或方向轉換無說明 |
| 2–4 | 無任何校外經驗 |

**技能標籤相關性：**
| 分數 | 標準 |
|------|------|
| ≥8 | 標籤精準對應職務內容且為可搜尋關鍵字 |
| 5–7 | 相關但摻雜軟實力標籤稀釋搜尋度 |
| 3–4 | 通用標籤或與職務不符 |
| 1–2 | 無標籤 |

**管理責任：** 若候選人聲稱資深/主管職位，卻無管理規模（人數、預算、範疇） → 紅旗，子項扣至 0。

**最終分：** `(成果×0.4 + 脈絡×0.2 + 標籤×0.2 + 管理×0.1 + JD×0.1) / 10 × 25`

Key constraints: 工作內容描述 ≤2000 chars/職位，工作技能 max 6 tags/職位。

## 求職條件 (Job Preferences) — 滿分 8

| 評估面向 | 標準（內部評分 0–10） |
|---------|-------------------|
| 薪資 vs 市場行情 | 9–10: 符合年資/學歷台灣市場行情，或「面議」有強佐證；5–7: 偏高但在合理討論範圍；2–4: 高出市場 30%+ 且無特殊技能支撐；1: 空白 |

> 薪資合理性判斷請參考 `references/salary-benchmarks.md`，對照候選人職類與年資。
| 期望職稱 vs 實際經歷 | 9–10: 職稱與年資相符；5–7: 略有落差但可解釋；2–4: 明顯不符（2年資歷申請 Director） |
| 期望工作內容 | 依是否有提供 JD 分兩條路徑（見下方） |

**期望工作內容評分（依情境）：**

有提供 JD 時：
- 9–10：針對 JD 闡述自己能帶來的具體價值（能力 ＋ 預期貢獻），與職位需求高度匹配
- 6–8：有內容但偏「我想要」，未充分呼應 JD 需求
- 3–5：空白 — 錯失展現職位匹配度的機會
- 1–2：內容方向與 JD 明顯相反

無 JD 時：
- 9–10：闡述自己能帶來的價值（能力導向、非願望導向），保有對話空間
- 6–7：空白 — 開放策略，保持機會廣度，為合理選擇
- 4–5：通用願望清單（「希望從事行銷工作」）— 自我設限卻無額外價值
- 1–3：過度具體且限制性強（特定產業 ＋ 特定城市 ＋ 特定型態並列）— 大幅縮窄機會

**最終分：** `平均三個面向分 / 10 × 8`

## 專長 (Expertise) — 滿分 15

**可信度交叉驗證（最重要）：**
工作經歷或專案成就中，是否有支持此專長的具體證據？
宣稱「數據分析專長」但全份履歷無任何資料工具出現 → 扣 1–2 分（可信度缺口）。

| 分數 | 標準 |
|------|------|
| ≥8 | 具體、可搜尋、符合業界術語，有跨區塊佐證 |
| 6–7 | 廣泛但合理，部分佐證 |
| 4–5 | 模糊或複合詞，缺乏佐證但方向正確 |
| 1–3 | 全是軟實力聲明，或極度通用無搜尋價值 |

**最終分：** `評分 / 10 × 15`

Key constraints: 專長描述 ≤1000 chars，專長特色標籤 max 6。

## 證照 (Certifications) — 滿分 4

| 分數（/4） | 標準 |
|-----------|------|
| 3.5–4 | 直接對應目標職位且有效；高價值證照 |
| 2.5–3.5 | 相關但部分過期或間接相關 |
| 1.5–2 | 有證照但與目標職缺關係較間接 |
| 0.5–1 | 有證照但與職務方向不符（至少展現學習意願） |
| 0 | 缺失（若職位明確要求則列為嚴重不足） |

**證照價值參考：**
| 高價值 | 中等價值 | 低附加值 |
|--------|---------|---------|
| PMP、CFA、CISSP、AWS/GCP 認證、律師/會計師/醫師執照、JLPT N1/N2、TOEIC 900+ | Google Analytics、Scrum Master、專業技術甲級、TOEIC 750–899 | 電腦基礎檢定、駕照（除非職務必要）、TOEIC <600 |

## 語文能力 (Language Skills) — 滿分 3

| 分數（/3） | 標準 |
|-----------|------|
| 2.5–3 | 程度真實，有佐證（TOEIC 分數、海外學習/工作、外商經歷）；無「三語全精通」 |
| 1.5–2.5 | 程度合理，無明顯過度聲稱，或有一定生活/工作語言使用脈絡 |
| 0.8–1.5 | 所有語言全填「精通」但無佐證（此為常見填法，給予基本分） |
| 0 | 目標職位明確需要特定語言但缺失 |

佐證加分信號：TOEIC/IELTS/JLPT/HSK 分數、國際公司工作、海外求學。

## 專案成就 (Projects & Achievements) — 滿分 8

| 評估維度 | 權重 | 標準（內部評分 0–10） |
|---------|------|-------------------|
| 個人貢獻清晰度 | 40% | 8–10: 角色與具體貢獻清楚；4–7: "參與"型，貢獻有說明但模糊；1–3: 只有專案名稱 |
| 量化影響 | 40% | 8–10: 明確前後指標與規模；6–8: 有數字但規模不清；2–5: 無數字但有具體描述；1: 純名稱 |
| 時效性與相關性 | 20% | 5 年以上舊案且無近期作品 → 扣 1–2 點；與目標職位不符 → 扣 1–2 點 |

加分：有 live demo、GitHub 連結或已發布作品 → +0.3（上限不超過 8）

**最終分：** `(貢獻×0.4 + 量化×0.4 + 時效×0.2) / 10 × 8`

Key constraints: 專案說明 ≤2000 chars。

## 自傳 (Autobiography) — 滿分 8

必須包含四個元素：
1. **背景與起點**（從哪裡來）
2. **核心能力**（能做什麼，附具體佐證）
3. **動機與目標**（為何選此職/公司，非套話）
4. **對公司的價值**（帶來什麼具體貢獻）

| 分數（/8） | 標準 |
|-----------|------|
| 7–8 | 四個元素齊全；有具體成就；針對公司/職位；無模板套語 |
| 5.5–6.5 | 三個元素；大體具體但含 1–2 段通用敘述 |
| 4–5.5 | 兩個元素；有基本內容，讀起來像工作經歷延伸但尚可 |
| 2.5–4 | 明顯套模板；有基本自我介紹，缺個人化 |
| 1–2 | 極短或幾乎空白，幾乎無參考價值 |

**紅旗（無論長度都降分）：**
- 以「我叫XXX，畢業於...」開頭
- 全篇無任何數字或成就
- 重複工作經歷（非補充，而是複述）
- 超過 3500 字的通用內容

Key constraints: 中文 ≤4000 chars，英文 ≤8000 chars。

| 分數（/3） | 標準 |
|-----------|------|
| 2.5–3 | 真正差異化內容（學術發表、開源貢獻、演講、重大獎項、有成效的側案）；含富媒體或有效連結 |

## 軟實力綜合評估 (Soft Skills Meta-Score) — 滿分 6

**此區塊不評估個別軟實力聲明，而是從整份履歷跨區塊評估三項指標。**

### 指標一：量化密度 (Quantification Rate) — 滿分 3

統計**工作經歷 + 專案成就**中，含有具體數字/指標/規模的描述條目佔總條目比例：

| 量化比例 | 分數（/3） |
|---------|----------|
| ≥70% | 3 |
| 50–69% | 2.5 |
| 30–49% | 1.8 |
| 15–29% | 1.0 |
| <15% | 0.5 |

輸出時請附：「共 X 條描述，其中 X 條含量化指標，量化比例 X%」

### 指標二：硬實力佔比 (Hard Skill Ratio) — 滿分 2

統計全履歷**所有技能標籤**（個人特色 + 工作技能 + 專長標籤），計算硬技能標籤佔比：

> **硬技能標籤定義：** 工具（Python、Figma）、業界術語（SEO、Scrum、B2B）、可驗證能力（資料庫設計）
> **軟技能標籤定義：** 個性描述（積極、負責、細心、溝通協調）

| 硬技能標籤比例 | 分數（/2） |
|-------------|----------|
| ≥60% | 2 |
| 40–59% | 1.5 |
| 20–39% | 1.0 |
| <20% | 0.5 |

輸出時請附：「共 X 個標籤，其中 X 個為硬技能，佔比 X%」

### 指標三：軟實力佐證質量 — 滿分 1

| 質量 | 分數（/1） |
|------|----------|
| 多數軟實力聲明有具體行為佐證（"帶領 5 人跨部門小組，3 個月完成 OO"） | 1 |
| 部分有佐證，部分裸聲明 | 0.5 |
| 全部裸聲明，無任何行為佐證 | 0.2 |

**軟實力綜合最終分：** 三指標加總（最高 6 分）
