# 產業別關鍵字與評分調整

當目標職缺屬於以下產業時，套用對應的關鍵字清單與評分側重調整。
若目標職缺未提供，不套用產業別調整（維持通用評分）。

---

## 如何使用

1. 根據用戶的目標職缺，判斷屬於哪個產業類別
2. 載入對應的關鍵字清單，用於 JD Gap 分析的「補充關鍵字」
3. 套用對應的「評分側重調整」說明，在 scoring.md 基礎上微調各維度

---

## 科技 / 軟體工程

**關鍵字清單（常見高價值關鍵字）：**
- 程式語言：Python, JavaScript, TypeScript, Go, Java, Rust, C++, Ruby, PHP
- 框架/平台：React, Vue, Angular, Node.js, Django, FastAPI, Spring Boot, Rails
- 雲端/基礎設施：AWS, GCP, Azure, Docker, Kubernetes, Terraform, CI/CD
- 資料：SQL, PostgreSQL, MongoDB, Redis, Elasticsearch, Kafka, Spark
- 方法論：Agile, Scrum, TDD, Code Review, System Design, Microservices
- 軟實力側重：Technical Leadership, Mentoring, Cross-functional collaboration

**評分側重調整：**
- 附件（GitHub/個人作品）：實際滿分視同 ×1.5（2分→3分效益）
- 專案成就：程式碼可看到的側案 > 工作內含專案
- 專長可信度：有 GitHub 提交記錄或開源貢獻為最高佐證
- 自傳：技術面試前不常被仔細閱讀，低於滿分 50% 不扣額外建議次數

---

## 數位行銷 / 電商

**關鍵字清單：**
- 工具：Google Analytics, GA4, Google Ads, Meta Ads, LINE Ads, HubSpot, Mailchimp
- 技術：SEO, SEM, CRM, A/B Testing, Conversion Rate Optimization (CRO)
- 指標：ROAS, CPA, CTR, CVR, LTV, Churn Rate, MQL, SQL
- 平台：Shopify, Shopline, MOMO, PChome, 蝦皮, LINE官方帳號
- 方法論：Growth Hacking, Content Marketing, Influencer Marketing, Retention Marketing

**評分側重調整：**
- 量化密度：行銷職比其他職能更容易量化，≥70% 才算高標（通用標準以此職能較嚴格）
- 個人特色標籤：純軟實力標籤（細心、積極）在行銷職中更常見，但仍應有 3 個以上工具/指標類標籤
- 附件：有 Portfolio / Campaign brief / 數據報告截圖加分

---

## 設計 / UX

**關鍵字清單：**
- 工具：Figma, Sketch, Adobe XD, Illustrator, Photoshop, After Effects, Principle
- 方法論：Design Thinking, User Research, Usability Testing, Information Architecture
- 技術：Prototyping, Wireframing, Design System, Accessibility (WCAG), Responsive Design
- 指標：Task Completion Rate, SUS Score, NPS, Click-through Rate

**評分側重調整：**
- 附件（作品集）：此職位附件實際滿分視同 ×3（缺少作品集視為嚴重缺口 ❌）
- 專案成就：說明中需包含「問題定義 → 設計過程 → 最終結果」三段結構
- 自傳：設計思維與使用者同理心的描述優先於職責列表

---

## 財務 / 會計

**關鍵字清單：**
- 工具：SAP, Oracle ERP, Excel (VBA/樞紐), Power BI, QuickBooks
- 技術：財務報表分析, 預算規劃, 成本控制, 稅務申報, 內部控制, IFRS
- 證照：CPA（會計師）, CFA, ACCA, 稅務記帳士, 證券分析師
- 指標：ROE, ROA, EBITDA, 現金流量, 負債比率

**評分側重調整：**
- 證照：此職能證照滿分視同 ×2（CPA/CFA 缺失且職位要求時，評分嚴格降低）
- 精確性語言：描述中「處理 X 筆交易」「管理 X 億預算」類數字比成效數字更有說服力
- 工作技能標籤：應以軟體工具+財務術語為主，通用軟實力標籤不計入有效標籤

---

## 人資 / 行政

**關鍵字清單：**
- 工具：Workday, BambooHR, 1111/104 系統, ADP, 薪資軟體（鼎新e系列）
- 技術：招募 (Talent Acquisition), 薪酬設計, 績效管理, 組織發展 (OD), 勞基法
- 指標：招募週期, 人才留任率, 員工滿意度 (eNPS), 培訓完成率

**評分側重調整：**
- 工作描述中「勞基法合規」「勞健保作業」類描述算充分職責佐證
- 管理責任：HR 職位「管理」意涵包含專案管理與流程設計，不侷限人員管理

---

## 業務 / 客戶成功

**關鍵字清單：**
- 工具：Salesforce, HubSpot CRM, LINE Business, Notion, Slack
- 技術：B2B Sales, B2C Sales, Key Account Management, Customer Success, Upsell/Cross-sell
- 指標：ARR, MRR, Quota Attainment, NRR, Deal Cycle Time, Win Rate, Churn Rate

**評分側重調整：**
- 量化密度：業務職是量化最容易的職能，≥60% 才算達標（通用標準為 ≥30%）
- 求職條件薪資：含浮動獎金的薪資結構合理，評分時底薪部分對照基準
- 工作經歷：無業績數字的業務履歷直接列為「嚴重缺口」❌
